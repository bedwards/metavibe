Robotics

Open Source Machines | December 2025

Robotics presents unique challenges for open research. Software can be shared instantly; hardware cannot. Yet the field has developed remarkable open-source infrastructure, from the Robot Operating System that runs on thousands of platforms to open hardware designs that anyone can build.

The Robot Operating System (ROS) is the backbone of this ecosystem. Kilted Kaiju is the latest ROS 2 short-term release (targeting Ubuntu 24.04 and Windows 10), while Jazzy Jalisco is the current LTS release. Recordings from ROSCon 2025 are now available, documenting the state of the art.

2025 Research Highlights

HyperGraph ROS

HyperGraph ROS, published in March 2025, presents an open-source robot operating system that unifies intra-process, inter-process, and cross-device computation into a computational hypergraph for efficient message passing and parallel execution.

The system dynamically selects optimal communication mechanisms while maintaining a consistent API. For intra-process messages, it uses Intel-TBB Flow Graph with C++ pointer passing (zero memory copying). Inter-process and cross-device communication seamlessly switches to ZeroMQ. Evaluation in diverse robotic scenarios demonstrates significantly higher transmission and throughput efficiency compared to ROS 2.

ARK Framework

ARK, published in June 2025, is an open-source, Python-first robotics framework designed to address the gap where current robot stacks demand steep learning curves, low-level C/C++ expertise, and fragmented tooling.

ARK ships with reusable modules for control, SLAM, motion planning, system identification, and visualization, along with native ROS interoperability. It presents a Gym-style environment interface that allows users to collect data, preprocess it, and train policies using state-of-the-art imitation-learning algorithms like ACT and Diffusion Policy.

The ROS Ecosystem

ROS 2 was redesigned from the ground up to meet challenges set forth by modern robotic systems. ROS 1 wasn't designed with many necessary production-grade features. ROS 2 addresses real-time requirements, security, and multi-robot coordination that production deployments demand.

Key Components

  • Navigation - Path planning and obstacle avoidance
  • MoveIt - Motion planning for manipulation
  • Gazebo - Physics simulation for testing
  • RViz - 3D visualization
  • tf2 - Coordinate frame transforms

Research Frameworks

PyRobot

PyRobot is an open-source robotics framework for research and benchmarking—a lightweight, high-level interface on top of ROS that provides consistent, hardware-independent mid-level APIs to control different robots. It abstracts away details about low-level controllers and inter-process communication.

Open Hardware

ROMR (ROS-based Open-source Mobile Robot) is a low-cost, modular mobile robot using off-the-shelf components, additive manufacturing, aluminum profiles, and consumer hoverboard motors. Fully compatible with ROS, with a 90kg payload capacity and under $1500 cost.

Open hardware designs like ROMR lower barriers to robotics research. Labs that couldn't afford commercial platforms can build their own, then share improvements back to the community.

Why Robotics Shares

Several factors push robotics toward openness:

Complexity. No one lab can build everything. Robots need perception, planning, control, simulation, and hardware. Open source lets labs specialize while sharing infrastructure.

Reproducibility. Claims about robot performance are hard to verify without access to the same software. Open code enables reproduction.

Standards. ROS became a de facto standard precisely because it was open. Proprietary alternatives couldn't achieve the same network effects.

Industry adoption. Companies like Amazon, Boston Dynamics, and NVIDIA build on ROS. Open source proved more reliable than proprietary middleware.

The Hardware Gap

Software sharing is easier than hardware sharing. A robot costs thousands or millions to build; code costs nothing to copy. This creates asymmetry—well-funded labs can test on real robots while others rely on simulation.

The field addresses this through better simulation (Gazebo, Isaac Sim), open hardware designs (ROMR, Open Manipulator), and cloud robotics that provide remote access to physical platforms. The gap remains, but it's narrowing.

The result is a field where the fundamental software infrastructure is free, where research papers typically include code, and where a student with a laptop can develop algorithms that later run on million-dollar robots.

claude-base

Foundation for Claude Code projects

A production-ready starting point for projects where Claude Code is a primary developer. Built on principles from the Anthropic Engineering Blog.

View on GitHub Read CLAUDE.md

Quick Start

# Clone claude-base (keep it for future projects)
git clone https://github.com/bedwards/claude-base.git ~/claude-base

# Create your new project
mkdir my-app && cd my-app
~/claude-base/bin/init.ts --name my-app

# Setup
npm install
cp .env.example .env
cp .secrets.example .secrets
npm run db:up && npm run dev

What's Included

TypeScript + Testing

ESLint, Vitest with coverage, Playwright E2E. No Jest.

Screenshot Framework

Capture and compare screenshots for visual verification. Claude reads these to guide UI/UX decisions.

GitHub Integration

Actions for CI/CD, PR checks, rate-limit-aware issue creation, Claude GitHub App integration.

Discord Communication

Tools for Claude instances to coordinate work and avoid conflicts on shared resources.

Postgres + Docker

Local development matches production. Migrations and seeds included.

Pre-commit Hooks

Blocks secrets from being committed. Runs typecheck, lint, and tests.

Core Principles

From Anthropic's Engineering Blog:

"Context, therefore, must be treated as a finite resource with diminishing marginal returns."
"A major failure mode occurred when agents tried to do too much at once—essentially to attempt to one-shot the app."
"Claude struggled to recognize end-to-end failures without explicit prompting to test as a human user would."

Our interpretation:

  • One feature at a time. Complete it. Verify it. Move on.
  • Screenshots are your eyes. Use them constantly.
  • No mock data in production. Fail fast, hard, and ugly.
  • If it looks like it works, it must actually work.

For Claude Instances

The CLAUDE.md file contains comprehensive instructions:

  • Own your PRs—don't wait for human review
  • Own the main branch—fix it if it breaks
  • Work in an infinite development loop
  • Take on big, hairy goals
  • Limit interactions with the human prompter

What's NOT Included

  • Significant example code to delete
  • Opinions about your domain logic
  • MCP servers or SKILLS
  • Vendor lock-in beyond standard tools
